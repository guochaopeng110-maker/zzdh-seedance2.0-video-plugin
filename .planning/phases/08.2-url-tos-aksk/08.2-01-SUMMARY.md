# Phase 08.2 Plan 01: 固定参数策略与安全治理 Summary

**已完成 V2 插件参数来源治理改造，核心 URL/TOS 来源固定，AK/SK 输入面收敛并补齐自动化验证。**

## Accomplishments

- `main.py` 增加 `_resolve_tos_credentials` 与 `_resolve_runtime_sources`，统一固定来源策略。
- `_sanitize_params` 不再让 `base_url` 驱动 create/query endpoint，legacy `base_url` 仅兼容读取并标记 deprecated。
- TOS 凭证治理改为 env 优先（`ZLHUB_TOS_AK`/`ZLHUB_TOS_SK`），缺失时回退固定策略；日志不打印密钥明文。
- `ui/index.html` 移除 `BaseUrl` 输入及保存逻辑，并增加“核心地址固定策略”提示。
- 新增 `tests/test_param_source_governance.py` 覆盖来源优先级、防漂移与日志脱敏场景。
