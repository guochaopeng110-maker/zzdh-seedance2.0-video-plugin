# Phase 08.3 Research: 数载接口适配策略

**Date:** 2026-04-27  
**Phase:** 08.3  
**Status:** Complete

## Research Question

在“参考旧版 zlhub 插件”的前提下，如何最小风险适配数载接口并满足“禁用 content 下载接口”约束？

## Findings

1. `video_plugin_zlhub_seedance` 主体编排（参数清洗、创建任务、轮询、下载、日志、任务库）可复用。
2. 旧插件审核链路（`AuditAESCipher` + 审核 API）与数载需求冲突，必须整体移除。
3. 数载文档以任务接口为核心，下载可直接使用轮询结果中的 `url`，与“禁用 `/v1/videos/{task_id}/content`”约束一致。
4. 为避免接口漂移，建议在 UI 与日志层明确写入“本插件不使用 content 下载端点”。

## Implementation Decision

1. 新建独立目录 `video_plugin_shuzai_seedance`，不改动原 zlhub 插件。
2. 任务接口固定为：
   - `POST /v1/video/generations`
   - `GET /v1/video/generations/{task_id}`
3. 下载逻辑只接受轮询返回的 `video_url/url`。
4. UI 删除审核项，补充端点约束提示。

## Risks and Mitigations

- 风险：第三方返回结构可能存在 `task_id/id` 差异。  
  缓解：创建任务时做多字段兜底解析。

- 风险：状态枚举可能存在别名差异。  
  缓解：状态归一化支持 `queued/running/succeeded/failed` 及常见同义词。

- 风险：误加回 content 下载路径。  
  缓解：把“禁止 content 端点”写入计划与验收检查，执行 `rg` 作为硬校验。
