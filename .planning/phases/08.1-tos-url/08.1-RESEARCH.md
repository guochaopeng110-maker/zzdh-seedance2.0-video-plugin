# Phase 08.1 Research: 参考素材上传TOS并转换公网URL

**Date:** 2026-04-23
**Phase:** 08.1
**Status:** Complete

## Research Question

如何在不改变宿主调用契约的前提下，让 `video_plugin_zlhub_seedance_V2` 支持：
- 输入本地文件路径/Base64 时自动上传 TOS 并替换为公网 URL；
- 审核接口和视频生成接口都只接收最终公网 URL/asset URL。

## Findings

### 1. 协议约束（来自 requires2 文档）
- 视频生成接口不支持 Base64 图片，要求先上传云存储拿公网 URL。
- 素材审核接口也要求参考图片是公网 URL。
- 因此“上传并转换URL”是插件内部必须补齐的前置步骤，而不是审核接口内部逻辑。

### 2. 现有代码差距
- `_call_material_audit_api` 仅接受 `http/https`，已拒绝本地/Base64。
- `_build_content_items` 依赖 `_normalize_media_url`，目前同样拒绝本地/Base64。
- 当前缺少“本地/Base64 -> TOS URL”的能力层。

### 3. TOS 接入要点
- 使用 TOS SDK 上传对象，返回可访问 URL 形如：
  `https://{bucket}.{endpoint}/{object_key}`
- 关键配置项：`AK/SK/REGION/BUCKET/ENDPOINT`。
- 需要通过插件参数或环境变量注入，不能硬编码到仓库。

## Implementation Implications

1. 在 `main.py` 新增 TOS 上传工具函数：
- `_build_tos_client(params)`
- `_upload_file_to_tos(local_file_path, params)`
- `_upload_data_url_to_tos(data_url, params)`
- `_normalize_or_upload_media_url(media_value, field_name, params)`

2. 在两个入口接入统一转换：
- 审核入口前：先转换 `reference_images`，再传 `_call_material_audit_api`。
- 生成 payload 前：`_build_content_items` 对 image/video/audio 调用统一转换函数。

3. UI 参数需要补齐：
- `tos_ak`, `tos_sk`, `tos_endpoint`, `tos_region`, `tos_bucket`。

4. 错误策略：
- 缺配置/上传失败/解析失败都抛 `PLUGIN_ERROR:::`。
- 错误文案包含“建议直接传公网 URL 或补齐 TOS 配置”。

## Validation Architecture

### Unit-level
- 输入为 `http/https`：原样返回。
- 输入为 `asset://`：原样返回。
- 输入为本地路径：调用上传函数并返回 URL。
- 输入为 `data:`：调用 data-url 上传并返回 URL。
- 输入非法字符串：抛 `PluginFatalError`。

### Integration-level
- 审核分支收到的 `images` 全是 `http/https`。
- 生成 payload 中 `content.*_url.url` 不再出现 `data:` 或本地路径。

### Regression-level
- 已存在的公网 URL 工作流行为不变。
- 审核后返回的 `asset://` 仍能用于后续视频生成。

## Risks and Mitigations

- 风险：上传凭证未配置，导致本地/Base64流程全部失败。
  - 缓解：在参数清洗阶段提前校验；只在需要上传时才强制校验凭证。

- 风险：Base64 文件过大导致内存和上传失败。
  - 缓解：增加上限校验，沿用已有 30MB 限制。

- 风险：上传成功但 URL 短时不可访问。
  - 缓解：明确提示重试；后续阶段可加 URL 探活。
