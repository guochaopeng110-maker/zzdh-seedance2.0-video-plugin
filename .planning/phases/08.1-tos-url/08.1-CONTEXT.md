# Phase 08.1: 实现参考素材TOS上传并在审核/生成前统一转公网URL - Context

**Gathered:** 2026-04-24
**Status:** Ready for planning

<domain>
## Phase Boundary

在 `video_plugin_zlhub_seedance_V2` 内明确并固化“参考素材预处理 + 审核/生成接口参数来源策略”：
1. 审核链路继续只接收公网 URL；本地文件与 `data:` 由插件先上传 TOS 后再参与审核或生成。
2. `TASK_CREATE_URL`、`TASK_QUERY_URL`、`ASSET_BASE_URL`、`TOS_ENDPOINT`、`TOS_REGION`、`TOS_BUCKET` 不从 `index.html` 输入，统一由 `main.py` 固定策略提供。
3. `tos_ak`、`tos_sk` 也不由前端配置输入，按本阶段固定值策略处理。
4. 不改变宿主契约（`generate(context)` 参数结构不变）。

</domain>

<decisions>
## Implementation Decisions

### 参数来源与固定策略
- **D-01:** `task_create_url` 与 `task_query_url` 由 `main.py` 内部统一计算/固定，不暴露到 `ui/index.html`。
- **D-02:** `asset_base_url`、`tos_endpoint`、`tos_region`、`tos_bucket` 由 `main.py` 固定值控制，不由前端输入覆盖。
- **D-03:** `tos_ak`、`tos_sk` 不从 `index.html` 读取，按阶段固定值处理：
  - `tos_ak = "AKLTNzI3MDM1NDQwZGMyNDM2M2E5YzY3MGE1ZGVjNzgyMzY"`
  - `tos_sk = "TmpFNFpUUmxaRE16TmpobE5EWTRaVGxrTXpabVpXRTJObVpsTVRjek1HWQ=="`

### 审核与生成接口一致性
- **D-04:** 审核提交接口固定走 `https://asset.zlhub.cn/api/asset/upload/async`，审核查询固定走 `https://asset.zlhub.cn/api/task/{task_id}`。
- **D-05:** 视频任务创建接口固定走 `https://api.zlhub.cn/v1/task/create`，任务查询固定走 `https://api.zlhub.cn/v1/task/get/{id}`。
- **D-06:** 每次 discuss/plan/execute 均需校验“代码实际调用 URL”与 `docs/requre2` 文档一致，防止常量与文档漂移。

### 素材预处理链路
- **D-07:** `_normalize_or_upload_media_url` 作为唯一入口，判定顺序保持：`asset://` -> `http/https` -> `data:` -> 本地路径 -> 非法格式报错。
- **D-08:** `_call_material_audit_api` 保持仅接收公网 URL；传入前必须完成预处理。
- **D-09:** `_build_content_items` 构建 payload 时对 image/video/audio 一律先走预处理。

### 可观测性与风险控制
- **D-10:** 日志中继续只输出 `tos_ak/tos_sk` 是否存在，不打印明文密钥。
- **D-11:** 错误仍统一 `PLUGIN_ERROR:::` 前缀，审核与上传失败必须携带可定位原因（缺参数、URL 不合法、上传失败等）。

### the agent's Discretion
- 在不改变固定参数策略前提下，可自由优化错误文案、重试次数与日志字段结构。
- 若后续需要从固定值切回可配置，作为新 phase 处理，不在本 phase 混入。

</decisions>

<specifics>
## Specific Ideas

- 本次讨论明确：`index.html` 仅承担必要业务参数输入，不承担审核/TOS/任务接口地址配置。
- 本阶段核心验收不是“可配置性”，而是“固定参数策略下审核接口与视频任务接口稳定可用且与文档一致”。

</specifics>
