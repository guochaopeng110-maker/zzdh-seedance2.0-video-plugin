# Phase 08.1: 实现参考素材TOS上传并在审核/生成前统一转公网URL - Context

**Gathered:** 2026-04-23
**Status:** Ready for planning

<domain>
## Phase Boundary

在 `video_plugin_zlhub_seedance_V2` 内补齐“素材URL预处理层”：
1. 输入若为本地文件路径或 Base64(data:)，先上传到 TOS，拿到公网 URL。
2. 输入若已是 http/https URL，直接透传。
3. 该预处理层同时服务于素材审核入口与视频创建 payload 构建入口。
4. 保持宿主调用契约不变（`generate(context)` 仍接收 `reference_images/videos/audios`）。

</domain>

<decisions>
## Implementation Decisions

### URL 预处理策略
- **D-01:** 新增统一函数（建议名：`_normalize_or_upload_media_url`），输入原始 media 值，输出最终可用 URL。
- **D-02:** 判定顺序固定：`asset://` -> `http/https` -> `data:` -> 本地路径 -> 其他非法格式。
- **D-03:** `asset://` 允许透传（用于审核后回填的资产ID）；`http/https` 允许透传。
- **D-04:** `data:` 与本地文件路径触发 TOS 上传，成功后替换为公网 URL。
- **D-05:** 上传失败必须抛 `PLUGIN_ERROR:::`，并包含可定位原因（缺配置/文件不存在/网络错误）。

### TOS 对接策略
- **D-06:** 参考 `docs/requre2/TOS临时公开云存储对接.md`，使用 TOS SDK 上传文件对象。
- **D-07:** 凭证从插件参数读取（`tos_ak/tos_sk/tos_endpoint/tos_region/tos_bucket`），不得硬编码在代码中。
- **D-08:** object key 统一放在 `images/` 前缀下，文件名使用 UUID 防冲突。
- **D-09:** Base64 输入先落地临时文件再上传，上传后清理临时文件。

### 审核与生成链路接入
- **D-10:** `_run_seedance_orchestration` 在审核触发前先对 `reference_images` 做 URL 预处理。
- **D-11:** `_build_content_items` 构建 payload 前对 image/video/audio 统一走 URL 预处理。
- **D-12:** 审核接口 `_call_material_audit_api` 保持“仅接受 http/https”不变；它接收的应已是预处理结果。

### the agent's Discretion
- Base64 MIME 类型与扩展名映射可采用最小可用策略（默认 `.bin`，常见图片 MIME 映射到 `.png/.jpg/.webp`）。
- 临时文件目录使用插件目录下 `tmp_uploads/`，避免污染工作区根目录。

</decisions>
