# Phase 8 Plan 01: 适配 requires2 视频与素材审核接口

## Objective

- **What:** 基于 Phase 7 已有链路，产出独立的 `video_plugin_zlhub_seedance_V2` 插件并完成 requires2 协议切换。
- **Why:** 当前实现仍依赖旧版 zlhub 任务地址与 AES 审核协议，不满足 `docs/requre2` 的新接口契约。
- **Output:** 可被宿主加载的 V2 插件，审核链路与视频生成链路均对齐新文档，且具备可追踪日志与最小回归验证。

<action>
在 V2 `main.py` 将 `_call_material_audit_api` 迁移为 requires2 模式：
1. 删除 `AuditAESCipher` 与 `encrypted_data` 请求体路径；审核请求体固定为 `{"images": [...], "asset_type": "Image", "callback_url": "..."}`。
2. 新增输入校验函数：`reference_images` 中每项必须 `http://` 或 `https://`；若为本地路径、空值或 `data:`，立即抛 `PLUGIN_ERROR:::不支持base64或本地文件，请提供公网URL`。
3. 提交使用 `POST https://asset.zlhub.cn/api/asset/upload/async`，请求头写入 `X-Access-Token`、`X-Track-Id`。
4. 查询使用 `GET https://asset.zlhub.cn/api/task/{task_id}`，默认间隔 3 秒，直到 `status=completed` 或超时。
5. 从 `result.items[*].downstream_asset_id` 组装 `asset://<id>`；若 `submit_review_status!=1` 或缺失 `downstream_asset_id`，抛出含 source_url 的失败错误。
6. 将 `audit_task_id`、`track_id`、审核条目数量写入 `_log_event` metadata。
</action>
